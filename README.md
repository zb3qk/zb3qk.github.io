# Workflow for Posting

## Create a file in '\_posts'
Follow the naming scheme as defined here: '2018-08-20-bananas.md', ie. 'date-name.md'.
The header portion of the Markdown file must be notated as follows:
```
---
layout: post
author: jill
---
```
Determining the layout (ie. formatting template) to follow and any other information necessary to fill out that template, for example: the author of the post.

# File System

## \_data
Data that will be used throughout the entire site, for example the headers for the navigation bar

## \_includes
HTML elements that are to be included in multiple palces in the site

## \_posts
Where the Blog posts live

## \_sass
Where the CSS and SASS files are referenced from

## \_site
The site that is compiled from the Jekyll format

## \_assets
Images, other css, or other information to be referenced in the site.
